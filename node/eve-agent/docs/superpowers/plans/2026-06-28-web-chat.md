# Web Chat Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give eve-agent a browser chat UI (local dev/self-test) that shows streamed assistant text, tool-call cards, and reasoning, talking to the built-in `/eve/v1/*` HTTP API.

**Architecture:** A custom channel (`agent/channels/web-chat.ts`) serves a single-page HTML (string constant in `agent/lib/web-chat-page.ts`). The page's vanilla JS calls same-origin `/eve/v1/session`, reads the NDJSON stream, and renders events. No new dependencies, no frontend framework.

**Tech Stack:** TypeScript (NodeNext), eve `defineChannel` / `GET` from `eve/channels`, vanilla browser JS (fetch + ReadableStream), NDJSON event stream.

---

## Verification strategy (read before implementing)

This project ships **no test runner** and the spec mandates **zero new dependencies**; the frontend logic lives inside an HTML string and cannot be unit-imported. Therefore this plan uses **end-to-end verification** instead of unit TDD:

- **Gate:** `npm run typecheck` must pass after every task.
- **Acceptance:** each task ends with a concrete `eve dev --no-ui` + browser/curl check mapped to a spec §2 success criterion.
- **Event-field caveat:** the eve docs describe event *semantics* but not exact JSON key names for every payload. Task 2 first prints a real event to the browser console to confirm field names before coding renderers against them. This is intentional observation-over-assumption.

**Commit policy:** per the user's global rule, do NOT auto-commit. Pause for explicit user confirmation before each `git commit`.

**Spec:** `docs/superpowers/specs/2026-06-28-web-chat-design.md`

## Implementation notes (deviations found during execution)

- **Channel route:** `GET("/chat")`, not `GET("/")` — eve's dev server owns `/` (returns its own landing page) and shadows a custom channel mounted at root. Confirmed via `/eve/v1/info`, which listed `urlPath: "/"` for `web-chat` but `GET /` still served eve's page.
- **Server port:** `2000` for this dev run (not `3000`); read it from the dev banner.
- **Event field names (measured from the live NDJSON stream):** `message.appended` → `data.messageSoFar` (cumulative) / `data.messageDelta` (increment); `message.completed` → `data.message` / `data.finishReason`; session POST body → `{ sessionId, continuationToken, ok }`. The plan body's earlier `cumulative`/`delta`/`text` guesses were wrong; the shipped code uses the real names.
- **Stream is long-lived:** the NDJSON stream does not EOF after a turn, so `openStream` keeps a single reader and relies on event types (`session.waiting` / `session.completed`) for turn boundaries instead of `reader done`.
- **Per-turn assistant bubble:** events carry no message id, so each `sendMessage` creates a fresh assistant bubble instead of reusing one by id.

Replace the entire `WEB_CHAT_HTML` string in `agent/lib/web-chat-page.ts` with:

```ts
export const WEB_CHAT_HTML: string = `<!doctype html>
<html lang="zh">
<head>
  <meta charset="utf-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1" />
  <title>eve-agent · Web Chat</title>
  <style>
    :root { color-scheme: light dark; --bubble-user: #2563eb; --bubble-assistant: #1f2937; }
    * { box-sizing: border-box; }
    body { margin: 0; font: 15px/1.5 system-ui, sans-serif; background: #0b1020; color: #e5e7eb; }
    main { max-width: 760px; margin: 0 auto; display: flex; flex-direction: column; height: 100vh; padding: 16px; gap: 12px; }
    header h1 { margin: 0; font-size: 18px; }
    header p { margin: 2px 0 0; color: #9ca3af; font-size: 12px; }
    #messages { flex: 1; overflow-y: auto; display: flex; flex-direction: column; gap: 10px; }
    .msg { padding: 10px 12px; border-radius: 10px; white-space: pre-wrap; word-break: break-word; }
    .msg.user { align-self: flex-end; background: var(--bubble-user); color: #fff; }
    .msg.assistant { align-self: flex-start; background: var(--bubble-assistant); }
    .msg .role { font-size: 11px; opacity: .6; margin-bottom: 4px; }
    #error { color: #fca5a5; background: #7f1d1d33; padding: 8px 10px; border-radius: 8px; }
    form { display: flex; gap: 8px; }
    input { flex: 1; padding: 10px; border-radius: 8px; border: 1px solid #374151; background: #111827; color: inherit; }
    button { padding: 10px 16px; border: 0; border-radius: 8px; background: var(--bubble-user); color: #fff; cursor: pointer; }
    button:disabled { opacity: .5; cursor: default; }
  </style>
</head>
<body>
  <main>
    <header>
      <h1>eve-agent</h1>
      <p>eve framework learning assistant · web chat</p>
    </header>
    <div id="messages"></div>
    <div id="error" hidden></div>
    <form id="chat-form">
      <input id="input" placeholder="Ask about the eve framework..." autocomplete="off" />
      <button id="send" type="submit">Send</button>
    </form>
  </main>
  <script>
    // --- 会话状态 ---
    const state = {
      sessionId: null,      // durable session id（首条 POST 响应拿到）
      continuationToken: null, // 续发凭证（每次响应更新）
      streamIndex: 0,       // 已消费的事件计数，用于 startIndex 续读，避免重复
      status: "ready",      // ready | streaming
    };

    const messagesEl = document.getElementById("messages");
    const errorEl = document.getElementById("error");
    const form = document.getElementById("chat-form");
    const input = document.getElementById("input");
    const sendBtn = document.getElementById("send");

    // --- 渲染辅助 ---
    function appendUser(text) {
      const el = document.createElement("div");
      el.className = "msg user";
      el.textContent = text;
      messagesEl.appendChild(el);
      messagesEl.scrollTop = messagesEl.scrollHeight;
    }

    function ensureAssistant(id) {
      // id 缺省时退化为单条助手消息
      const domId = "m-" + (id || "assistant");
      let el = document.getElementById(domId);
      if (!el) {
        el = document.createElement("div");
        el.id = domId;
        el.className = "msg assistant";
        const role = document.createElement("div");
        role.className = "role";
        role.textContent = "assistant";
        const text = document.createElement("div");
        text.className = "text";
        el.appendChild(role);
        el.appendChild(text);
        messagesEl.appendChild(el);
      }
      messagesEl.scrollTop = messagesEl.scrollHeight;
      return el.querySelector(".text");
    }

    function showError(message) {
      errorEl.textContent = message;
      errorEl.hidden = false;
    }
    function clearError() { errorEl.hidden = true; errorEl.textContent = ""; }

    function setStatus(next) {
      state.status = next;
      const busy = next === "streaming";
      input.disabled = busy;
      sendBtn.disabled = busy;
    }

    // --- 发送消息 ---
    async function sendMessage(text) {
      clearError();
      appendUser(text);
      setStatus("streaming");

      // 首条走 /eve/v1/session；续发走 /eve/v1/session/<id> 并带 continuationToken
      const url = state.sessionId
        ? "/eve/v1/session/" + encodeURIComponent(state.sessionId)
        : "/eve/v1/session";
      const body = state.sessionId
        ? { continuationToken: state.continuationToken, message: text }
        : { message: text };

      let res;
      try {
        res = await fetch(url, {
          method: "POST",
          headers: { "content-type": "application/json" },
          body: JSON.stringify(body),
        });
      } catch (err) {
        showError("Network error: " + err.message);
        setStatus("ready");
        return;
      }
      if (!res.ok) {
        showError("HTTP " + res.status);
        setStatus("ready");
        return;
      }
      const data = await res.json().catch(() => ({}));
      if (data.sessionId) state.sessionId = data.sessionId;
      if (data.continuationToken) state.continuationToken = data.continuationToken;

      await openStream();
    }

    // --- 读取 NDJSON 事件流 ---
    let abortController = null;
    async function openStream() {
      // 取消上一次未完成的读取，避免重叠
      if (abortController) abortController.abort();
      abortController = new AbortController();

      const url = "/eve/v1/session/" + encodeURIComponent(state.sessionId)
        + "/stream?startIndex=" + state.streamIndex;
      let res;
      try {
        res = await fetch(url, { signal: abortController.signal });
      } catch (err) {
        if (err.name !== "AbortError") showError("Stream error: " + err.message);
        return;
      }
      if (!res.ok || !res.body) { showError("Stream HTTP " + res.status); return; }

      const reader = res.body.getReader();
      const decoder = new TextDecoder();
      let buffer = "";
      while (true) {
        const { value, done } = await reader.read();
        if (done) break;
        buffer += decoder.decode(value, { stream: true });
        let nl;
        while ((nl = buffer.indexOf("\\n")) >= 0) {
          const line = buffer.slice(0, nl).trim();
          buffer = buffer.slice(nl + 1);
          if (!line) continue;
          let event;
          try { event = JSON.parse(line); }
          catch { continue; } // 跳过不完整的行
          handleEvent(event);
        }
      }
    }

    // --- 事件分发（v1：仅文本与会话生命周期）---
    function handleEvent(event) {
      const d = event.data || {};
      switch (event.type) {
        case "message.appended": {
          // 增量：优先用 cumulative（累计文本），否则在 delta 上累加
          const t = ensureAssistant(d.id);
          if (typeof d.cumulative === "string") t.textContent = d.cumulative;
          else if (typeof d.delta === "string") t.textContent += d.delta;
          break;
        }
        case "message.completed": {
          const t = ensureAssistant(d.id);
          if (typeof d.text === "string") t.textContent = d.text;
          break;
        }
        case "session.waiting":
        case "session.completed":
          setStatus("ready");
          break;
        case "step.failed":
        case "turn.failed":
        case "session.failed":
          showError((d.message || event.type));
          setStatus("ready");
          break;
        default:
          break; // 其余事件类型在 Task 3/4 处理
      }
      state.streamIndex += 1;
    }

    // --- 绑定输入 ---
    form.addEventListener("submit", (e) => {
      e.preventDefault();
      const text = input.value.trim();
      if (!text || state.status === "streaming") return;
      input.value = "";
      void sendMessage(text);
    });
  <\/script>
</body>
</html>`;
```

> Note: the `<\/script>` escape is required because this is a JS template literal; an unescaped `</script>` would close the host string prematurely.
