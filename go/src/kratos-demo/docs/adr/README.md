# Architecture Decision Records

This directory stores ADRs (Architecture Decision Records) for the kratos-demo project.

## When to write an ADR

An ADR is written only when **all three** conditions are met:

1. **Hard to reverse** — the cost of changing your mind later is meaningful
2. **Surprising without context** — a future reader will wonder "why did they do it this way?"
3. **The result of a real trade-off** — there were genuine alternatives and you picked one for specific reasons

## ADR format

Each ADR is a markdown file named `NNNN-title-with-dashes.md`, following the standard template:

- **Title** — `# NNNN: Title`
- **Status** — proposed / accepted / deprecated / superseded
- **Context** — the situation that prompted the decision
- **Decision** — what was chosen
- **Alternatives considered** — genuine options that were evaluated
- **Consequences** — trade-offs accepted
